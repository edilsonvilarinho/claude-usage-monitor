Seção 5
5 Janelas de Sessão (Session Windows)

O resumo de janelas mostra as sessões de 5h. Cada linha exibe: data/hora de início, estado (Aberta/Fechada), e valor em %.

Janela 1 Aberta 13 abr 10:00 → em andamento 73% pico 14:30
Janela 2 Fechada 12 abr 14:00 → 19:00 68%
Janela 3 Fechada 12 abr 08:00 → 14:00 100%
Regras de exibição (v9.x): • Janela Aberta: exibe peak (pico atual) — apenas se peak > 0 ou final > 0
• Janela Fechada: exibe final (valor no momento do fechamento)
• Correção de bug: antes mostrava pico (100%) ao fechar, agora mostra valor final (73%)
• sessionWindowCount: número de janelas abertas no dia (default 1)
• sessionAccum: soma dos picos das janelas completadas

Seção 9
9 Estados Especiais
⏱
Rate Limit (429)
API retornou 429. Backoff exponencial: 5min × 2^(n-1), cap 10min. Persiste entre restarts.
❌
Erro de API (5xx/timeout)
Servidor indisponível ou timeout. Retry 3× com backoff interno. Depois, erro genérico com backoff próprio.
🔐
Credenciais (401)
Token expirado (401) ou arquivo não encontrado. Abre modal de credenciais para login.
🔄
Reset Detectado
Janela de 5h resettou (resets_at avançou ≥30min). Notificação opcional + atualiza sessionWindows.
Seção 10
10 Credenciais e Autenticação

Source of Truth

O sistema nunca solicita API key. Credenciais são lidas diretamente do arquivo gerado pela CLI:

  • ~/.claude/.credentials.json — caminho primário
  • \\wsl.localhost\<distro>\home\<user>\.claude\.credentials.json — fallback WSL

Seleção: usa arquivo com mtime mais recente (token mais provável de estar válido).

Renovação de Token

Margem: 5 minutos antes da expiração.
Fluxo: credentials.json → refreshToken → POST /v1/oauth/token → novo accessToken + expiresAt

Se renovação falhar (ex: sem internet), usa o token existente mesmo que próximo do vencimento. A autenticação falha silenciosamente no pollingService.

Seção 11
11 Smart Schedule (Agenda Inteligente)

Status do Sistema

🟢 VERDE — Modo Livre 🟡 ÂMBAR — Próximo do límite 🟣 ROXO — Otimizador Pré-Sessão 🔴 VERMELHO — Crítico

O sistema avalia em cascata: 1º VERDE (fora do expediente), 2º ROXO (otimização pré-sessão), 3º ÂMBAR (próximo do limite), 4º VERMELHO (crítico).

Parâmetros

activeDays0-6 (0=Dom)
workStart/workEndHorário trabalho (min)
breakStart/breakEndIntervalo (min)
sessionThreshold0-100% (default 80%)
weeklyThreshold0-100% (default 80%)
Seção 12
12 Cloud Sync

Arquitetura

Servidor self-hosted (Node.js/Express). Autenticação via JWT. Push/pull automático a cada 10 min ou manual.

Merge: union por chave — nenhum dado local é deletado por pull remoto. Maior mtime prevalece.

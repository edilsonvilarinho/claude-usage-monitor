Seção 5
5 Barras de Progresso: Sonnet e Créditos Extras

Além dos dois gauges principais, o popup exibe barras horizontais para Sonnet 3.5 (janela semanal) e créditos extras. Cada barra é calculada a partir do payload da API.

Sonnet
52%
Calculado via seven_day_sonnet.utilization × 100  ·  janela independente de 7 dias
Créditos Extras
28%
Calculado via (extra_usage.used_credits / extra_usage.monthly_limit) × 100
Quando a barra Sonnet aparece
Sempre que o payload contém seven_day_sonnet com utilization definido. Mostra uso do modelo Sonnet especificamente, separado do limite global.
Quando a barra Créditos aparece
Apenas se extra_usage.is_enabled === true e monthly_limit > 0. Usuários sem plano de créditos não veem essa barra.

Seção 8
8 Sistema de Notificações

O notificationService usa debounce para evitar re-notificações excessivas. Cada alerta tem condição de disparo, condição de reset e configuração independente.

Alerta de Sessão
Dispara quando: sessionPct ≥ sessionThreshold (padrão 80%)
Debounce: não re-notifica até sessionPct < resetThreshold (padrão 50%)
Configurável: toggle on/off + som ativável via settings
Alerta Semanal
Dispara quando: weeklyPct ≥ weeklyThreshold (padrão 80%)
Debounce: não re-notifica até weeklyPct < resetThreshold (padrão 50%)
Configurável: independente do alerta de sessão
Reset de Janela (5h)
Dispara quando: resets_at muda ≥ 1 hora (nova janela detectada)
Sem debounce — evento pontual, não acumula
Configurável: notifyOnWindowReset (padrão: true)
Queda de Uso
Dispara quando: uso cai abaixo de resetThreshold após alerta ter sido enviado
Reset do debounce: zera a flag de "já notificou"
Configurável: notifyOnReset (padrão: false)
Thresholds configuráveis
Threshold Padrão Range Persistência
sessionThreshold 80% 1 – 100 config.json
weeklyThreshold 80% 1 – 100 config.json
resetThreshold 50% 1 – 99 config.json
Seção 9
9 Janelas de Sessão (5h) e Detecção de Reset

A janela de sessão da Anthropic dura 5 horas. Várias janelas podem ocorrer no mesmo dia. O sistema detecta cada transição e acumula os dados.

Exemplo: 3 janelas no mesmo dia
08:00
Janela 1  ·  peak 65%
13:00
reset
13:00
Janela 2  ·  peak 88%
18:00
reset
18:00
Janela 3  ·  atual 40%
em progresso → reset em 23:00
Detecção de nova janela
|newResetsAt − storedResetsAt| ≥ 30 minutos
Evita falsos positivos por flutuação da API.
Ao detectar nova janela
sessionAccum += completedWindow.peak
sessionWindowCount++
maxSession reseta para a nova janela
Histórico
8 dias de dados mantidos (splice automático).
Cada entrada: { date, peak, resetsAtMinute }
Deduplicação
Por minuto: resets_at_minute = floor(resetsAt / 60000)
Evita duplicar a mesma janela em polls consecutivos.
Seção 10
10 Credenciais OAuth (credentialService)

O app lê o token de acesso diretamente do arquivo de credenciais da CLI do Claude. Nenhuma senha é armazenada; o CLI gerencia a autenticação inicial.

Fluxo de leitura e refresh
1
Lê o arquivo de credenciais
~/.claude/.credentials.json
No Windows com WSL: tenta também caminhos /mnt/c/Users/… e \\wsl$\…
2
Extrai e verifica o token
Lê claudeAiOauth.accessToken e compara expiresAt com Date.now()
3
Refresh automático se necessário
Se expira em < 5 min: POST console.anthropic.com/v1/oauth/token com grant refresh_token
4
Atualiza expiração
expiresAt = Date.now() + expires_in × 1000
Grava de volta no arquivo
Estrutura do arquivo
{
  "claudeAiOauth": {
    "accessToken": "sk-ant-oat01-…",
    "refreshToken": "…",
    "expiresAt": 1712345678000,
    "scopes": ["…"],
    "subscriptionType": "pro"
  }
}
Fallback WSL: No Windows, o app tenta detectar instâncias WSL e lê o arquivo .credentials.json do sistema de arquivos Linux montado, permitindo uso sem instalar o CLI nativo do Windows.
Seção 11
11 Persistência com electron-store

Todos os dados do app são persistidos via electron-store (JSON no sistema de arquivos). Quatro stores separados, cada um com schema validado.

Arquivo Localização Conteúdo principal
config.json %APPDATA%\Claude Usage Monitor\ theme, language, windowSize, sessionThreshold (80%), weeklyThreshold (80%), resetThreshold (50%), notifications (enabled/sound), autoRefresh, pollIntervalMinutes, cloudSync settings
accounts.json mesmo diretório usageHistory, dailyHistory (8 dias), timeSeries (por data), sessionWindows[], currentSessionWindow, rateLimitedUntil, rateLimitCount
cloud-sync.json mesmo diretório jwt (criptografado), jwtExpiresAt
sync-outbox.json mesmo diretório operações de sync pendentes (enviadas quando o servidor ficar disponível)
Dev vs Prod
Dev: %APPDATA%\Electron\config.json
Prod: %APPDATA%\Claude Usage Monitor\config.json
Regra de schema
Nunca apertar minimum/maximum em campos existentes sem uma migration — o electron-store crasha no startup se o valor salvo violar o schema.
